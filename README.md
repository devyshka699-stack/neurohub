# AI-услуги — web-витрина + AI backend + Telegram-бот (Модули 1–3)

Сайт-витрина услуг на нейросетях: каталог, заказ с загрузкой файла, оплата переводом на карту или криптовалютой, личный кабинет клиента и админ-панель. Заказы выполняются автоматически бесплатными нейросетями (модуль 2). Дополнительный канал продаж — Telegram-бот с оплатой Stars и реферальной системой (модуль 3).

## Стек

- **FastAPI** + **Jinja2** — бэкенд и шаблоны
- **SQLite** (через SQLAlchemy) — база данных, файл `shop.db` создаётся автоматически
- **Tailwind CSS** (CDN) — дизайн без сборки
- **AI backend**: очередь задач на `asyncio.Queue`, автоопределение модели по услуге, кэширование результатов (папка `ai_cache/`)

## Запуск локально

```bash
cd ai-services-shop
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Сайт: http://127.0.0.1:8000

При первом запуске автоматически создаются:

- 7 услуг каталога
- администратор: `admin@example.com` / `admin123` (переопределяется переменными `ADMIN_EMAIL` / `ADMIN_PASSWORD`)

Все настройки читаются из файла **`.env` в корне проекта** (переменные окружения имеют приоритет). Заполните значения и перезапустите сервер — изменения применятся автоматически, включая смену пароля админа.

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `SHOP_NAME` | название магазина (шапка, футер, заголовки) | `AI-услуги` |
| `SUPPORT_CONTACT` | контакт поддержки (@username или email), показывается в футере | пусто |
| `SECRET_KEY` | ключ подписи сессий | `dev-secret-change-me` |
| `ADMIN_EMAIL`, `ADMIN_PASSWORD` | учётка администратора — синхронизируется при каждом запуске | `admin@example.com` / `admin123` |
| `PAYMENT_CARD_NUMBER` | номер карты для перевода | заглушка |
| `PAYMENT_CARD_HOLDER` | имя получателя | заглушка |
| `PAYMENT_CARD_BANK` | банк | `Т-Банк` |
| `PAYMENT_CRYPTO_USDT_TRC20` | адрес USDT (TRC-20); пусто — валюта скрыта | пусто |
| `PAYMENT_CRYPTO_BTC` | адрес BTC; пусто — валюта скрыта | пусто |

Если оба криптоадреса пусты, вариант «Криптовалюта» не показывается в форме заказа. **Обязательно замените заглушки реквизитов и `SECRET_KEY` перед публикацией.**

## AI backend (модуль 2)

Каждой услуге соответствует своя нейросеть — всё бесплатное и работает локально (MacBook Air M3, 24 ГБ хватает):

| Услуга | Модель | Что нужно для работы |
|---|---|---|
| Тексты | Llama через **Ollama**, фолбэк **Groq API** | `ollama serve` + `ollama pull llama3.2`, либо `GROQ_API_KEY` |
| Изображения | Stable Diffusion через **ComfyUI** | ComfyUI на `localhost:8188` с чекпоинтом SD 1.5; русские промпты автопереводятся через Ollama |
| Удаление фона | **rembg** (u2net) | ставится из requirements, работает сразу |
| Озвучка | **Edge TTS** | ставится из requirements, нужен интернет |
| Колоризация | **ComfyUI img2img** (основной), **DeOldify** (запасной) | ComfyUI запущен; опционально `DEOLDIFY_DIR` |
| Апскейл | **Real-ESRGAN** (ncnn-vulkan) | бинарник, `REALESRGAN_BIN` |
| Логотипы | Stable Diffusion + **vtracer** (SVG) | ComfyUI; один значок по центру (не лист вариантов) |

Как устроено:

1. **Очередь** — `asyncio.Queue` с фоновым воркером внутри процесса FastAPI (`app/ai/queue.py`). После рестарта зависшие задачи возвращаются в очередь.
2. **Автовыбор модели** — тип задачи определяется по названию услуги (`app/ai/tasks.py`).
3. **Кэш** — ключ SHA-256 от типа задачи + описания + байтов входного файла. Повторный одинаковый заказ отдаётся из `ai_cache/` мгновенно.
4. **QC** — после генерации результат проверяется (`app/qc/`). Для аудио нужен `ffmpeg` (`FFMPEG_BIN` или PATH / imageio-ffmpeg).
5. **Запуск** — автоматически после кнопки «Подтвердить оплату» (отключается `AI_AUTO_PROCESS=0`) или вручную кнопкой «🤖 Выполнить через AI» в админке.
6. **Ошибки** — если модель недоступна (не запущен ComfyUI), заказ помечается «ошибка AI» с инструкцией в админке, и его можно выполнить вручную.

Переменные окружения AI: `OLLAMA_URL`, `OLLAMA_MODEL`, `GROQ_API_KEY`, `GROQ_MODEL`, `COMFYUI_URL`, `COMFYUI_CHECKPOINT`, `EDGE_TTS_VOICE`, `REALESRGAN_BIN`, `FFMPEG_BIN`, `DEOLDIFY_DIR`, `AI_AUTO_PROCESS`.

### Тексты на Render (Groq)

На бесплатном Render нет Ollama. Чтобы автогенерация текстовстов работала на проде:

1. Создайте ключ на [console.groq.com](https://console.groq.com) → API Keys (бесплатно).
2. На Render → сервис **neurohub** → **Environment** → добавьте:
   - Key: `GROQ_API_KEY`
   - Value: `gsk_...`
3. **Save, rebuild, and deploy**.

Локально ключ не обязателен — тексты идут через Ollama. Не коммитьте ключ в git и не кладите его в публичный чат.

### Контракт локального воркера (Мака) для заказов с Render

Сайт на Render и нейросети на Маке — разные машины. Воркер на Маке забирает оплаченные заказы и гоняет их через локальный AI.

**На Render (Environment):**

| Переменная | Значение |
|---|---|
| `REMOTE_WORKER` | `1` — не крутить AI на сервере, только ставить в очередь |
| `WORKER_TOKEN` | длинный секрет (тот же, что в локальном `.env`) |

**На Маке (`.env` рядом с проектом):**

| Переменная | Значение |
|---|---|
| `WORKER_BASE_URL` | `https://neurohub-hjs6.onrender.com` |
| `WORKER_TOKEN` | тот же секрет |
| `REMOTE_WORKER` | `0` (локальный сайт на :8017 по желанию обрабатывает сам) |
| `WORKER_POLL_INTERVAL` | `30` (секунды) |

**Запуск воркера** (ComfyUI/Ollama должны быть доступны, если нужны услуге):

```bash
cd ai-services-shop
.venv/bin/python -m app.worker
```

**API** (заголовок `Authorization: Bearer <WORKER_TOKEN>`):

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/api/worker/orders/pending` | Список заказов `in_progress` без результата |
| POST | `/api/worker/orders/{id}/claim` | Занять заказ (`ai_status=processing`) |
| GET | `/api/worker/orders/{id}/file` | Скачать файл клиента |
| POST | `/api/worker/orders/{id}/result` | Залить результат + QC → `review` или `done` |
| POST | `/api/worker/orders/{id}/error` | Зафиксировать ошибку AI |

Зависший `processing` дольше `WORKER_STALE_MINUTES` (по умолчанию 45) снова отдаётся в pending. Воркер раз в цикл дергает главную страницу Render, чтобы бесплатный инстанс не «спал» во время работы.

## Telegram-бот (модуль 3)

Бот работает в одном процессе с сайтом: общая SQLite-база, общая AI-очередь, результаты приходят прямо в чат.

Запуск: создайте бота у [@BotFather](https://t.me/BotFather), включите inline-режим (`/setinline`) и inline feedback (`/setinlinefeedback` → Enabled), затем:

```bash
export TELEGRAM_BOT_TOKEN=123456:ABC...
uvicorn app.main:app
```

Возможности:

- **/start** — каталог услуг кнопками, оформление заказа в диалоге (описание → файл → оплата)
- **Приём файлов** — фото или документ до 20 МБ прикрепляется к заказу
- **Оплата Telegram Stars** — цена в рублях автоматически пересчитывается в ⭐ (курс `TG_STARS_RATE_RUB`, по умолчанию 1.8₽/⭐); после оплаты заказ сразу уходит в AI-очередь
- **Inline-режим** — `@ваш_бот описание задачи` в любом чате → бот предлагает сгенерировать текст или картинку, кнопка ведёт к оформлению
- **Результат в чат** — картинки, аудио, текст и архивы бот присылает сам; если AI не справился, клиент получает сообщение, что заказ выполнит оператор (и получит результат после ручного выполнения в админке)
- **Рефералка** — /ref выдаёт ссылку `t.me/бот?start=ref_N`; после первой оплаты друга рефереру начисляется 50₽ (`REFERRAL_BONUS`) на баланс, балансом можно оплатить заказ целиком
- **/orders, /balance** — история заказов и баланс

Заказы из Telegram видны в общей админке с пометкой «из Telegram».

## Auto-marketing engine (модуль 4)

Движок привлечения клиентов: мониторит фриланс-биржи и Telegram-чаты, находит подходящие заказы, формирует персонализированные отклики и генерирует SEO-лендинги.

> ⚠️ **Важно про безопасность и легальность.** Автоматический постинг откликов на биржи и в чужие чаты нарушает правила большинства площадок и быстро приводит к бану аккаунтов. Поэтому движок по умолчанию работает в режиме **dry-run**: он собирает лиды и готовит тексты откликов, но **не отправляет** их. Отправку нужно включать осознанно и только там, где у вас есть на это право (свой аккаунт, разрешённые площадки). Сбор публичных данных и SEO — безопасны и включены.

Рубильники (переменные окружения):

| Переменная | Смысл | По умолчанию |
|---|---|---|
| `MARKETING_ENABLED` | включить планировщик | `0` (выключен) |
| `MARKETING_DRY_RUN` | не отправлять отклики, только готовить | `1` (dry-run) |
| `MARKETING_INTERVAL` | период проверки, секунды | `300` (5 мин) |
| `MARKETING_MIN_REPLY_INTERVAL` | антиспам: пауза между откликами на площадку | `60` |
| `MARKETING_TG_CHATS` | публичные Telegram-чаты для мониторинга (через запятую) | — |
| `PUBLIC_BASE_URL` | адрес сайта для sitemap и ссылок | `http://127.0.0.1:8000` |
| `PORTFOLIO_URL` | ссылка на портфолио в откликах | = `PUBLIC_BASE_URL` |

Составные части:

- **Парсеры бирж** (`app/marketing/sources/`) на Playwright: Kwork, Work-Zilla, YouDo, Avito Услуги. Ищут заказы по ключевым словам «нужен текст», «убрать фон», «озвучить видео» и т.д. Селекторы вынесены в начало каждого файла — вёрстка бирж меняется, правятся там.
- **Telegram-чаты** — мониторинг публичной веб-версии (`t.me/s/<чат>`) без авторизации и клиентского API.
- **Ключевые слова и матчинг** (`keywords.py`) — сопоставляют текст заказа с одной из 7 услуг; есть и общие сигналы «кто может сделать», «нужен дизайнер».
- **Шаблоны откликов** (`templates.py`) — «Сделаю за 30 минут. Цена: X. Портфолио: ссылка», с вариациями вступления/концовки под тип задачи, чтобы отклики не были идентичными.
- **Антиспам-лимитер** (`limiter.py`) — не более одного отклика на площадку за `MARKETING_MIN_REPLY_INTERVAL`.
- **Планировщик** (`engine.py`) — фоновый цикл каждые 5 минут: собрать → сматчить → сохранить новые (дедуп по площадка+id) → сформировать отклик → (в LIVE-режиме) отправить с учётом антиспама.
- **Дашборд лидов** — в админке `/admin/leads`: список найденных лидов со статусами, готовыми текстами откликов и кнопкой «Прогнать цикл сейчас».
- **SEO-генератор** (`seo.py`) — 56 лендингов под ключевые запросы («убрать фон с фото бесплатно», «сгенерировать картинку онлайн», «озвучка текста нейросеть» и т.д.), каждый ведёт на форму заказа нужной услуги, с FAQ и микроразметкой Schema.org. Доступны на `/uslugi` и `/uslugi/<slug>`, добавлены в `/sitemap.xml` и `/robots.txt`.

Настройка Playwright (один раз, для парсеров бирж):

```bash
.venv/bin/playwright install chromium
```

Без установленного браузера парсеры бирж просто вернут пустой результат (движок не упадёт), а SEO и матчинг продолжат работать.

## Client retention system (модуль 5)

Превращает разовых клиентов в постоянных: промокоды, накопительные скидки, сегментация и автоматические рассылки. Сообщения доставляются в Telegram (если клиент пришёл из бота) и всегда дублируются в личном кабинете как уведомления.

Что происходит автоматически:

- **После выполнения заказа** — клиент получает промокод на скидку 30% на следующий заказ, действует 48 часов (`PROMO_POST_ORDER_PERCENT`, `PROMO_POST_ORDER_HOURS`).
- **Неактивность 7 дней** — win-back: «У нас появилась новая услуга: генерация логотипов. Скидка 50% для старых клиентов» + персональный промокод (`WINBACK_DAYS`, `WINBACK_PERCENT`). Повтор не чаще раза в `WINBACK_COOLDOWN_DAYS` дней.
- **5 оплаченных заказов** — клиент становится VIP: персональный менеджер и скидка 20% навсегда (`VIP_ORDERS_THRESHOLD`).
- **Накопительная скидка** — 5 заказов → 10%, 10 → 20%, 20 → 30%. Применяется автоматически при оформлении.
- **Ежемесячная рассылка** — «Новые возможности нейросетей в этом месяце» раз в `NEWSLETTER_EVERY_DAYS` дней тем, кто хоть раз заказывал.

Скидки и промокоды:

- При оформлении заказа берётся **лучшая** из двух скидок (не суммируются): накопительная скидка клиента или скидка по промокоду. Итоговая цена сохраняется в заказе (`base_price`, `discount_percent`, `final_price`) и учитывается везде — на сайте, странице оплаты и в Telegram (включая пересчёт в Stars).
- Промокоды одноразовые, привязаны к клиенту, с датой истечения. Видны в личном кабинете.

Сегментация клиентов (`retention/segments.py`): новый / активный / постоянный / VIP / уходящий — по числу оплаченных заказов и давности последнего.

Планировщик (`retention/scheduler.py`) запускается вместе с приложением (`RETENTION_ENABLED=1`) и раз в час (`RETENTION_INTERVAL`) прогоняет win-back и рассылку. Вручную цикл запускается кнопкой «Прогнать рассылки удержания» в админке.

Переменные окружения: `RETENTION_ENABLED`, `RETENTION_INTERVAL`, `PROMO_POST_ORDER_PERCENT`, `PROMO_POST_ORDER_HOURS`, `WINBACK_DAYS`, `WINBACK_PERCENT`, `WINBACK_COOLDOWN_DAYS`, `VIP_ORDERS_THRESHOLD`, `NEWSLETTER_EVERY_DAYS`.

## Quality control & auto-correction (модуль 6)

Каждый результат AI проверяется перед отправкой клиенту (`app/qc/`). Заказ проходит цепочку: **генерация → автопроверка → автокоррекция (перегенерация) → подтверждение человеком → доставка**.

Проверки по типу задачи (балл 0–100, порог `QC_MIN_SCORE`, по умолчанию 70):

- **Тексты** (pyspellchecker): орфография (мягкий сигнал — русский словарь неполный), уникальность (доля уникальных слов, повторяющиеся фразы), соответствие запросу (пересечение ключевых слов), достаточный объём.
- **Изображения** (Pillow): файл открывается, размер не меньше 64px, пропорции не «полоска» (до 3:1), не однотонное (детект брака генерации — критично, сразу непроход), не размыто (дисперсия краёв).
- **Озвучка** (pydub): аудио не пустое, не тишина (dBFS), длительность соответствует объёму текста (~11 симв/сек, допуск в обе стороны). Без ffmpeg деградирует до проверки по размеру файла.

Автокоррекция: при балле ниже порога результат перегенерируется до `QC_MAX_ATTEMPTS` раз (по умолчанию 3) **с другими параметрами** — для текстов растёт temperature и ужесточается промпт, для изображений — новый seed и больше шагов диффузии. Если ни одна попытка не прошла, сохраняется лучшая — заказ всё равно уходит на ручную проверку с пометкой «низкое качество».

Финальная проверка человеком: после пройденной автопроверки заказ получает статус **«На подтверждении»** (а не сразу «Выполнен»). В админке виден QC-отчёт (балл, метрики, проблемы, число попыток) и кнопки: **«Подтвердить и отправить»** (доставка клиенту + промокод удержания), **«Перегенерировать»**, замена результата вручную. Если у админа привязан Telegram, приходит уведомление о заказе, ждущем подтверждения. В кэш попадают только прошедшие проверку результаты.

Переменные окружения:

| Переменная | По умолчанию | Что делает |
|---|---|---|
| `QC_ENABLED` | `1` | автопроверка качества включена |
| `QC_MIN_SCORE` | `70` | минимальный балл для прохождения |
| `QC_MAX_ATTEMPTS` | `3` | попыток генерации при низком качестве |
| `QC_REQUIRE_APPROVAL` | `1` | ждать подтверждения человеком; `0` — отправлять клиенту сразу после автопроверки |

## Financial dashboard (модуль 7)

Дашборд доходов для админа: `/admin/finance` (ссылка «Финансы» в шапке). Все суммы считаются по **оплаченным** заказам (статусы «В работе», «На подтверждении», «Выполнен») по цене со скидкой.

Что показывает:

- **Выручка за сегодня / неделю / месяц** и число заказов за каждый период.
- **Средний чек** — за всё время и отдельно за текущий месяц.
- **Самая популярная услуга** — по числу оплаченных заказов, с выручкой.
- **Самый прибыльный канал привлечения** — Сайт / Telegram-бот / Реферальная программа, по выручке.
- **График роста доходов** (Chart.js): выручка по дням за 30 дней (столбцы) + накопительный итог (линия), плюс графики выручки по услугам и «пончик» по каналам.
- **Прогноз на месяц** — run-rate: выручка месяца ÷ прошедших дней × дней в месяце.

Обновление: статистика кэшируется на час (`FINANCE_CACHE_SECONDS`, по умолчанию 3600), страница сама перезагружается раз в час (`<meta refresh>`). Кнопка «Пересчитать сейчас» сбрасывает кэш вручную.

## Как это работает

1. Клиент регистрируется, выбирает услугу и заполняет форму заказа (описание, файл, контакт, способ оплаты).
2. На странице оплаты показываются реквизиты; после перевода клиент нажимает «Я оплатил(а)» — заказ переходит в статус «Проверка оплаты».
3. Админ в панели `/admin` подтверждает оплату (статус «В работе»), затем нажимает «Выполнить», прикладывая файл результата и комментарий.
4. Клиент видит статус и скачивает результат в личном кабинете `/cabinet`.

## Деплой на Render.com (бесплатный тир)

1. Залейте проект в репозиторий GitHub.
2. На [render.com](https://render.com) → **New → Web Service** → подключите репозиторий.
3. Настройки:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
4. В **Environment** добавьте `SECRET_KEY`, `ADMIN_PASSWORD` и реквизиты оплаты.

Нюанс бесплатного тира: диск эфемерный — SQLite и загруженные файлы стираются при передеплое/перезапуске. Для постоянного хранения подключите Persistent Disk (платно) или используйте PythonAnywhere.

## Деплой на PythonAnywhere (бесплатный тир)

1. Зарегистрируйтесь на [pythonanywhere.com](https://www.pythonanywhere.com), откройте Bash-консоль:

```bash
git clone <ваш-репозиторий> ai-services-shop
cd ai-services-shop
python3.10 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. **Web → Add a new web app → Manual configuration → Python 3.10.**
3. Укажите virtualenv: `/home/<логин>/ai-services-shop/.venv`.
4. В WSGI-файле (FastAPI — это ASGI, поэтому на бесплатном тире нужен адаптер `a2wsgi`):

```bash
pip install a2wsgi
```

```python
# содержимое WSGI-файла
import sys
sys.path.insert(0, '/home/<логин>/ai-services-shop')

from a2wsgi import ASGIMiddleware
from app.main import app

application = ASGIMiddleware(app)
```

5. Нажмите **Reload**. SQLite и загруженные файлы на PythonAnywhere сохраняются между перезапусками — для этого проекта это удобнее бесплатного тира Render.

## Структура проекта

```
ai-services-shop/
├── app/
│   ├── main.py        # маршруты FastAPI
│   ├── models.py      # User, Service, Order, PromoCode, Lead + статусы
│   ├── database.py    # подключение SQLite
│   ├── config.py      # реквизиты и настройки (env)
│   ├── security.py    # хэширование паролей (PBKDF2)
│   ├── seed.py        # начальные услуги и админ
│   ├── tgbot.py       # Telegram-бот (модуль 3)
│   ├── ai/            # AI-очередь, кэш, обработчики (модуль 2)
│   ├── marketing/     # авто-маркетинг + SEO-страницы (модуль 4)
│   ├── retention/     # удержание клиентов (модуль 5)
│   ├── qc/            # контроль качества результатов (модуль 6)
│   └── templates/     # Jinja2 + Tailwind
├── uploads/           # файлы клиентов и результаты (создаётся сама)
├── requirements.txt
└── shop.db            # SQLite (создаётся сама)
```
